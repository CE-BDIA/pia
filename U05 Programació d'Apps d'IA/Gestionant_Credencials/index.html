    Gestió de credencials

    Des dels nostres programes haurem d'accedir a serveis externs, al qual devem d'estar registrats i per tant tindre unes credencials, però, com posem eixes credencials als nostres programes.

    El que no hem de posar mai son coses com:

    Python
    1
2
3
    usuari="middle"
password="T0mm@"
token="rj8539cdea89_Ff"

    Posar les credencials directament al codi com en l'exemple que has proporcionat pot ser problemàtic per diverses raons:

    1. Manca de Seguretat: fa que sigui fàcil per a algú que tingui accés al codi veure i utilitzar aquestes credencials.

    2. Dificultat per a la Gestió de Credencials: És difícil de canviar o actualitzar-les quan sigui necessari. Si les credencials canviessin (per exemple, per raons de seguretat), hauries de modificar manualment el codi, el que pot ser propens a errors i costós en termes de temps (hard-coded).

    3. Problemes de Manteniment: Les credencials poden estar disperses per tot el codi i és difícil de seguir-les.

    4. Risc d'Exposició Accidental: Si el teu codi s'ha de compartir o publicar, hi ha un risc d'exposar les credencials accidentalment. Això pot ocórrer, per exemple, si compartissis el codi en un repositori públic de GitHub sense adonar-te que les credencials estan incloses al codi.

    Per evitar aquests problemes, és millor utilitzar variables d'entorn o altres mètodes segurs.

    Utilitza Variables d'Entorn:

    La millor manera de manejar dades sensibles com ara contrasenyes en el teu codi de Python és utilitzant variables d'entorn. Això et permet emmagatzemar les dades de manera segura fora del teu codi i fer referència a elles des del codi.

    Aquí tens una manera senzilla de fer-ho:

    1. Defineix les variables d'entorn amb les teves credencials:
    Bash
    1
2
    export NOM_USUARI="el_teu_usuari"
export CONTRASENYA="la_teva_contrasenya"
    1. Des del teu codi de Python, pots accedir a aquestes variables d'entorn utilitzant la llibreria os:
    Python
    1
2
3
4
5
6
    import os

nom_usuari = os.environ.get("NOM_USUARI")
contrasenya = os.environ.get("CONTRASENYA")

# Ara pots utilitzar 'nom_usuari' i 'contrasenya' en el teu codi
    1. Quan s'executa el codi, les variables d'entorn es carregaran i el teu codi les utilitzarà sense haver de fer-ho hard-coded dins del teu script.

    2. Utilitza la Llibreria python-dotenv:

    El problema de l'anterior mètode és que hem de fer les exportacions de les variables d'entorn cada cop, i això implica en cada execució, sobretot en terminals distintes. Recorda que les variables d'entorn es reinicien quan es tanca la sessió, cosa que ocorre al tancar la terminal.

    Amb aquesta llibreria el que hem de fer és:

    • Crear a un arxiu (sempre amb el mateix nom) .env una col·lecció de línies variable=valor
    • La llibreria transfereix dites definicions a la terminal d'execució.
    • Ja podem carregar les variables desde l'entor an programa amb la llibreria os de la mateixa manera que abans

    Això fa que siga molt fàcil d'utilitzar i integrar.

    Exemple d'ús d'Arxiu .env
    Bash
    1
2
    NOM_USUARI=el_teu_usuari
CONTRASENYA=la_teva_contrasenya
    Python

    import os
from dotenv import load_dotenv

# Carrega les variables d'entorn des del fitxer .env
load_dotenv()

# Obté les credencials dels variables d'entorn
nom_usuari = os.getenv("NOM_USUARI")
contrasenya = os.getenv("CONTRASENYA")

# Ara pots fer servir 'nom_usuari' i 'contrasenya' al teu codi
print("Nom d'usuari:", nom_usuari)
print("Contrasenya:", contrasenya)
    Molta atenció
    • Mantén el fitxer .env fora del control de versions per evitar exposar les credencials.
    • No compartisques les credencials amb altres persones i mantén-les segures.
    • Actualitza les credencials regularment per mantenir la seguretat.
